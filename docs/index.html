Chapter 01 · Business

An intelligent platform for customs clearance

An intelligent platform that improves accuracy before clearance, reduces manual effort, and supports faster, more consistent customs processing. It helps teams work with greater confidence, lower the risk of errors, and move shipments through the clearance process more smoothly.

Problem Statement

Why this needs to exist

Businesses lose time and face compliance risk because product classification is manual, inconsistent, and often incomplete before customs submission. Existing tools are often siloed by carrier, shipper, or workflow, making it difficult to manage classification consistently across the business.

Issue 01
Slow clearance

Manual review and back-and-forth between shippers, agents, and customs holds shipments at the border. Every vague description or missing field triggers another round trip.

Issue 02
Higher compliance risk

Inaccurate or incomplete HS code data exposes businesses to penalties, reclassification, and rejected declarations — with costs that compound across shipments.

Issue 03
Fragmented workflows

Classification tools are siloed by carrier, shipper, or workflow. Teams can't scale a consistent classification practice across clients, channels, and business units.

Target Customer

Who we serve

The product serves three segments across the customs value chain. Each has a different operating model, a different urgency, and a different mix of product modes — so packaging and go-to-market adapt to the segment while the backend stays shared.

Segment A
Supply chain & logistics companies
Teams handling high shipment volumes that need faster, more consistent classification and customs documentation across many orders
What they care about
  • Operational efficiency and repeatability at scale
  • Visibility across shipments and teams
  • Keeping unit cost flat as shipment volume grows
Primary modes
  • Boost — refine partial or supplier-provided codes
  • Validate — catch mismatches before Bayan submission
Boost · Validate
Segment B
Ecommerce businesses
Businesses shipping cross-border into KSA that need scalable classification at checkout or order processing to reduce clearance delays
What they care about
  • Automation at scale across large catalogs
  • Product data that is customs-ready from day one
  • Fewer clearance surprises for the end customer
Primary modes

Segment C

Agents managing customs filings for multiple clients who need fast, accurate, and repeatable HS classification with less manual effort

Solution Overview

An AI platform for faster ZATCA clearance

Clear AI is an AI-powered customs classification platform built to help businesses pass ZATCA clearance faster by generating, improving, and validating HS code data before submission. It solves the classification workflow in three ways — create compliant classifications from scratch, improve classifications that are close but not optimal, and check whether code, description, and value make sense together before filing.

Three product modes

The product works in three modes depending on the quality of the input data and the stage of the workflow. This keeps it modular and practical rather than one-size-fits-all — users engage with the mode that matches what they have in hand.

Mode 01 · Create
Generate
Use when there is no usable classification. The system creates a ZATCA-ready HS code and compliant description from product data.
When to use
  • New SKUs with only a product name or short description
  • Sparse catalog data from suppliers or marketplaces
  • First-time classification for a client or product line
From scratch
Mode 02 · Improve
Boost
Use when a code already exists but could be more accurate. The system suggests a better classification and improves over time through AI learning.
When to use
  • Supplier-provided codes that look approximate
  • Legacy catalogs needing refinement at scale
  • Reclassification after ruling or regulation changes
Improve existing
Mode 03 · Check
Validate
Use when the code must be checked before submission. The system verifies whether HS code, description, and declared value are consistent and flags mismatches.
When to use
  • Final pre-submission check before Bayan
  • Screening declarations for suspicious combinations
  • Audit trail of why a classification was accepted
Pre-submission check
📌
Outcome-first framing: The business outcome — faster ZATCA clearance with lower compliance risk — is the headline. The product mechanics (Generate, Boost, Validate) are the support structure underneath.

Success Metrics by Mode

How we know it's working

Each product mode is measured against a small set of focused metrics, plus three overall product metrics that track business outcome.

Generate

Quality
Accepted without correction
Target → % of generated classifications accepted without manual correction

Measures whether Generate output is trusted end-to-end, not just directionally useful.

Speed
Time to first valid HS code
Target → average time from input to first valid HS code output

Covers latency plus any required input enrichment to reach a usable code.

Compliance
ZATCA-ready output rate
Target → % of Generate outputs formatted and structured for ZATCA without rework

Measures whether Generate output flows straight into downstream submission.

Boost

Lift
Improved over baseline
Target → % of classifications improved compared to the input code

Measures whether Boost is actually adding classification value beyond what the user had.

Efficiency
Reduction in manual reclassification
Target → time saved vs manual reclassification effort

Tracks Boost’s ability to replace slow, manual review cycles in large catalogs.

Accuracy
Accuracy uplift over existing codes
Target → measured uplift in classification accuracy vs existing codes

Validates that “improved” codes are actually more correct, not just different.

Validate

Detection
Invalid entries flagged
Target → % of invalid or suspicious entries flagged before submission

Measures whether Validate catches the problems it is supposed to catch.

Outcome
Reduction in clearance issues
Target → drop in clearance issues caused by inconsistent data

Links Validate output directly to real ZATCA clearance outcomes.

Workflow
Review resolution time
Target → median time to resolve flagged items in the review queue

Tracks whether flags are actionable and reviewable at operational speed.
