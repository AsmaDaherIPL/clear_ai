Chapter 01 · Business

Product definition for Clear AI

Why this product needs to exist, who it serves, what ships in V1, and what V2 unlocks. The working brief for product, engineering, and go-to-market alignment.

Problem Statement

Why this needs to exist

Pain 01
Incomplete product data stalls classification

Importers rarely provide complete product descriptions. Agents must manually research HS codes for vague inputs — a slow, error-prone process that directly delays clearance at Saudi borders.

Pain 02
Manual classification is a bottleneck at scale

Carriers processing high shipment volumes have no scalable way to classify goods automatically. Every item requires human effort, creating a linear relationship between volume and cost.

Pain 03
Existing tools are siloed to single carriers

Automation tools that do exist are internal to specific logistics providers. Independent clearance agents — who serve multiple carriers and importers — have no access to intelligent tooling.

Pain 04
ZATCA compliance adds layers of complexity

Saudi-specific requirements — Arabic descriptions, ZATCA XML format, and additional regulatory certificates — are not handled by generic global classification tools, forcing manual compliance work on every shipment.

Goals

What success looks like

Goals are split between what users gain and what the product needs to prove at MVP. Each goal is measurable — not a direction, but an outcome.

User Goals
  • Clearance agents can classify a product and generate a ZATCA-ready draft in under 30 seconds
  • Carriers can submit bulk manifests and receive classified outputs without manual intervention per item
  • Low-confidence results are surfaced clearly so human review effort goes only where it's needed
  • Any user — carrier or independent agent — can access the same capability without carrier lock-in
Product Goals
  • Demonstrate that AI classification meets or exceeds manual accuracy on a real Saudi shipment dataset
  • Ship a working API that a carrier integration can go live on within the MVP window
  • Cover all P0 features — including bulk processing and declaration generation — in V1
  • Establish ZATCA direct submission as a validated path for V2, not an assumption

📌
Non-Goals (V1): Clear AI will not handle duty payment, broker licensing, physical customs inspection coordination, or outbound export declarations in this release.
V1 Scope & Inputs

V1 Scope

V1 is a classification and review product, not a full customs automation engine. It supports one workflow: take a product description, suggest an HS code, show the reasoning, and let a human accept or override it.

V1 Does
  • Classify single items from text input
  • Return top HS candidates with a confidence score
  • Explain the decision using HS headings, notes, and rules
  • Flag low-confidence or ambiguous cases for human review
  • Produce a clearance-ready draft output for downstream use
V1 Does Not
  • Submit directly to ZATCA
  • Handle duty payment or broker licensing
  • Manage export declarations
  • Replace human review on uncertain items
  • Depend on OCR — text input only unless file intake is added later
📌
Product intent: Clear AI does not try to replace customs expertise. It helps humans move faster, reduces obvious errors, and makes the hard cases easier to review.

Supported Inputs in V1

✓  Accepted
  • Product name
  • Product description
  • Material
  • Brand / model
  • Existing client-provided HS code
  • Item category
  • Optional Arabic text if available

All input is user-entered text. The system is designed to work with whatever the user has — not require a complete data set.

✕  Not in V1
  • OCR from scanned invoices or packing lists
  • Image extraction or photo upload
  • Automatic PDF parsing
  • Full document ingestion pipeline

If file-based input is added later, OCR and document extraction become useful. For V1, text input is sufficient and simpler to validate.

Target Market

Who we sell to

Three addressable segments in the KSA customs value chain — distinct economics, deal shapes, and GTM motions. Same backend, differentiated packaging per segment. V1 lands in Segment A (beachhead) and opens the door to B and C via the same API.

Segment A · Beachhead
Carriers & 3PLs
Logistics companies, freight forwarders, and integrators clearing shipments into KSA at volume
Why this segment first
  • Highest shipment volume per account — classification cost scales linearly today
  • Existing TMS integrations mean API adoption path is well understood
  • Naqel is the design-partner anchor — proves scale before broader launch
Value drivers
  • Cost-per-classification reduction vs. manual agent effort
  • Clearance SLA improvement — faster ZATCA acceptance
  • Compliance risk reduction on HV declarations
GTM motion

Enterprise sales. API + admin console. Annual contract, volume-tiered pricing per classified line item. Design-partner discount for V1 anchors.

API · Web Console
Segment B · Expansion
Independent Clearance Agents
Licensed customs brokers serving multiple importer clients — SMB segment, carrier-agnostic
Why this segment
  • Underserved — carrier-internal tools don't reach them
  • Fragmented market; low-touch self-serve acquisition is viable
  • Low CAC unlocks healthy unit economics at SMB price points
Value drivers
  • Competitive parity with carrier-embedded tooling
  • Per-client submission history for audit and retention
  • Faster turnaround → more clients served per agent
GTM motion

Product-led / self-serve. Web UI subscription. Flat monthly tier with usage allowance + overage. Content + broker-association partnerships as top-of-funnel.

Web UI · Self-serve
Segment C · Long-term
Importers & Shippers
Businesses shipping goods into KSA — procurement, e-commerce, and enterprise import teams
Why this segment
  • Largest in headcount, smallest in per-account value
  • Best reached embedded in carrier workflow, not direct
  • Provides distribution leverage for Segment A carrier deals
Value drivers
  • Landed-cost visibility pre-booking reduces clearance surprises
  • Lower-friction classification = higher carrier conversion
  • Handoff-ready output for their clearance agent
GTM motion

Indirect. Lightweight self-serve estimator for awareness; primary monetisation via carrier-embedded classification at booking. Not a V1 direct-sales target.

Embedded · Self-serve
📈
Sequencing rationale: Segment A gives us reference logos, real-world accuracy data, and the revenue base to underwrite R&D. Segment B opens a lower-ACV but higher-margin self-serve motion on the same infrastructure. Segment C is a distribution play, not a standalone revenue line in V1–V2.
Features & Requirements

What we're building

V1 ships the core classify-to-declaration pipeline. Everything else is documented for architectural awareness — not a delivery commitment.

🔗
API-first principle: The web UI is a client of the same REST API. Every feature accessible in the UI is also accessible programmatically. No capability is UI-only.

Description
Release

Produces ZATCA-compliant Bayan declaration drafts in Arabic and English. Includes HS code, description, and duty rate reference.
V1
Confidence Flagging
Every result carries a confidence score. Below a configurable threshold, items are automatically routed to a human review queue.

Success Metrics

How we know it's working

Leading indicators measurable within weeks of launch. Lagging indicators tracked at 90 days.

Classification Quality
HS Code Accuracy Rate
Target → ≥ 92% on a held-out Saudi shipment dataset

Measured against expert-verified classifications. Baseline established during validation before launch.

Efficiency
Time to Classification
Target → < 1s per item (API), < 10 min per bulk batch (500 items)

Measured at p95 under expected load. Bulk jobs run async — clock starts at submission, ends at webhook delivery.

Review Load
Human Review Rate
Target → < 15% of items flagged for review

High-confidence classifications should handle the majority. Tracks model confidence calibration over time.

Adoption
API Integration Rate
Target → ≥ 1 live carrier integration within 60 days of launch

Primary B2B adoption signal. Tracked by first real production call from a non-test API key.

Compliance
Declaration Acceptance Rate
Target → ≥ 95% of generated declarations accepted by ZATCA without re-submission

Lagging indicator. Requires carriers to share ZATCA outcome data. Tracked at 90 days post-launch.

Retention
Agent Weekly Active Use
Target → ≥ 60% of registered agents active in week 4

Indicates Clear AI has replaced — not supplemented — the agent's current workflow for routine classifications.

Version 2

What V2 unlocks

V2 closes the loop from classification to submission, expands input modalities, and introduces multi-tenant access.

V2 Adds
  • Direct ZATCA submission with acknowledgement return
  • ZATCA status tracking + webhooks
  • Multi-tenant authentication and tenant-scoped history
  • Invoice / packing list OCR — auto-extract product data
  • Operations dashboard for review queue + accuracy trends
V2 Still Does Not
  • Handle duty payment or broker licensing
  • Manage outbound export declarations
  • Replace human review on low-confidence items
  • Offer mobile-native agent apps
🧭
V2 intent: Move Clear AI from a classification assistant to a closed-loop submission engine — with audited outcomes flowing back into accuracy tracking.
